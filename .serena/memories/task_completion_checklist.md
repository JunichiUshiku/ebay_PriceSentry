# タスク完了時チェックリスト

## コード変更後の確認事項

### 1. ビルド確認
```bash
cd frontend
npm run build
```
- [ ] ビルドがエラーなく完了すること
- [ ] TypeScriptの型エラーがないこと

### 2. Lint確認
```bash
npm run lint
```
- [ ] ESLintエラーがないこと
- [ ] 警告は可能な限り解消

### 3. 動作確認
```bash
npm run dev
```
- [ ] 開発サーバーが正常に起動すること
- [ ] 変更した機能が期待通り動作すること
- [ ] コンソールにエラーがないこと

### 4. UI確認
- [ ] レスポンシブデザインが維持されていること
- [ ] ダークモード対応（必要な場合）
- [ ] アクセシビリティが維持されていること

## ドキュメント更新

### AGENTS.md更新が必要な場合
大きな設計変更があった場合は、AGENTS.mdを更新：
- [ ] アーキテクチャ変更
- [ ] 新しいページ/機能の追加
- [ ] 技術スタック変更
- [ ] ワークフロー変更

### 設計ドキュメント（docs/）
- [ ] 新機能の設計はドキュメント化
- [ ] 既存設計との整合性確認

## AI運用5原則の遵守確認
- [ ] 第1原則: AGENTS.mdを参照し最新ルールを理解
- [ ] 第2原則: 大きな更新があればAGENTS.mdを更新
- [ ] 第3原則: チャット冒頭で5原則を出力済み
- [ ] 第4原則: 専門用語はわかりやすく説明
- [ ] 第5原則: ルールを厳守

## コミット前チェック
```bash
# ビルドとLintを一括実行
npm run build && npm run lint
```
- [ ] 不要なconsole.logを削除
- [ ] コメントは必要最小限
- [ ] 機密情報が含まれていないこと
