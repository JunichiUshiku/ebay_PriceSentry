# eBay Price Sentry UIデザイン決定メモ

作成日: 2026-05-03  
対象モック: `docs/mock-ui/index.html`  
元設計: `docs/superpowers/specs/2026-04-30-ebay-price-sentry-design.md`

## 1. デザイン方針

採用方針は **Dense Classic**。

- Navyサイドバー + 白背景 + Blueアクセント
- 業務管理ツールとして、装飾よりも一覧性と操作性を優先
- デスクトップは高密度テーブル
- iPhoneではテーブルではなく、タップしやすい高密度リスト
- 角丸は控えめ、カード半径は最大8px程度
- ステータスは色だけでなく、ドット・テキスト・行背景でも識別する

## 2. ナビゲーション

メニューは設計書に合わせて以下の4つのみ。

```text
ダッシュボード
承認待ち
設定
ログ
```

独立した「リスティング」メニューは作らない。  
リスティング関連の操作は以下に集約する。

- ダッシュボードのリスティング行
- 行クリック時の詳細モーダル
- 詳細モーダル内の設定タブ
- 設定画面のローカル設定一覧タブ

## 3. 共通レイアウト

### デスクトップ

- 左サイドバー: 168px
- トップバー: 44px
- メインコンテンツ: KPI、フィルタ、テーブルを縦に配置
- テーブル行クリックで詳細モーダルを開く
- 検索バーはトップバーに置かず、必要な場合はテーブル内検索で扱う

### iPhone

- 左サイドバーは非表示
- 下部に4メニューのボトムナビ
- KPIは上位4件のみを小さく表示
- 一覧はカード型ではなく、1行型の高密度リスト
- 1行の高さは約44px
- 390x844相当の表示で、下部ナビに隠れない範囲に約15件表示できる密度

20件表示案も検討したが、行高30pxではタップしづらいため不採用。  
現時点では **15件程度見える密度とタップしやすさのバランス** を採用する。

## 6. 価格ランキング

設計書どおり、価格ランキングは **合計価格の安い順でトップ10** を表示する。

自分の順位がトップ10外の場合は、トップ10の下に別ブロックで「あなた #N」を表示する。

### 列構成

現在の採用列。

```text
順位
セラーID
商品価格
送料
合計
差
```

以前は採用ライバルを示す `★` 専用列を置いていたが、情報量に対して列を1つ使うのはもったいないため削除した。

### 採用ライバルの表現

採用ライバルは、価格計算の基準になった競合商品。  
専用列ではなく、行全体の控えめなハイライトで表現する。

```text
背景: 薄緑
左端: 細い緑ライン
セラーID: 緑系の太字
```

現在のモックでは `#1 SellerDealsDude` が採用ライバル。

### 横幅

現状のランキングテーブルは固定列幅ではなく、ブラウザの `table-layout: auto` による可変幅。

現行モックの目安。

```text
テーブル最小幅: 620px
セラーID列: 約157px
```

今後、モーダル幅をさらに縮める場合は、セラーID列を120〜140px程度に抑え、長いセラーIDは省略表示する。
